5. 3D App Deeper Understanding Statement with Feedback from Testing

Deeper Understanding

This project goes beyond the base lab tasks in several areas:

  • Original concept and models. The lab Coca-Cola theme was replaced with an original bakery and food concept. Three original Blender models were created from scratch: a cheeseburger, a green apple and a bread basket. None of the lab assets were reused.
  • Advanced Blender workflows. The cheeseburger cheese layer was shaped using a cloth and collision simulation rather than manual modelling. The apple uses a procedural material node setup with UV baking so the material appears consistently in the browser. The bread basket was built using Extra Curve Objects, Knots/Celtic Links curves and NURBS ring shapes to produce the woven detail. Baked texture images are used across all models so the Blender shader work transfers correctly to the Three.js viewer without recreating shader networks in the browser.
  • Web performance optimisation. The bread basket was decimated using Blender's Decimate modifier to reduce polygon count while keeping the woven detail readable, because the original file caused noticeable slowdown in the browser, particularly when wireframe mode was enabled.
  • Custom JavaScript animation system. A lightweight tween helper was written from scratch using requestAnimationFrame timing and time-based progress values mapped through easing functions, rather than using a third-party animation library. This was used for the burger layer drop animation, the apple roll animation and all camera transitions.
  • Model-specific animations. The burger animation targets named GLB parts individually, dropping each layer into place with bounce easing and timed pop sound effects. The apple rolls side to side to reflect how an apple moves on a surface. The bread basket rotates to show its woven form from all sides. These interactions are tied to the story of each food object rather than being a single repeated behaviour.
  • Smooth camera transitions. Four named camera viewpoints (Front, Side, Top, Detail) move the camera and OrbitControls target together using tweened interpolation, so the transition feels controlled rather than snapping suddenly to a new position.
  • GitHub codebase. The full codebase, exported GLB models and original Blender source files are published to a GitHub repository as required for the additional going-beyond marks.

Feedback from Testing

Functional testing was carried out on the Gallery page to check that model selection, wireframe mode, animations, lighting presets, camera viewpoints, music and navigation all worked as expected across different screen sizes.

A classmate also completed a cognitive walkthrough of the main controls, which identified several issues that were then fixed:

  • Apple animation clipping. The apple rolling animation originally moved into the scene display tray. The tray size was reduced to give the model room to roll without clipping.
  • Bread basket performance. The wireframe view of the bread basket was slow and less smooth due to the high polygon count. The Decimate modifier was applied in Blender to reduce complexity while keeping the woven shape readable.
  • Button icon inconsistency. Emoji-style icons on control buttons did not display correctly across all browsers and devices. The icons were removed and clear text labels were kept instead, which also improved accessibility.
  • Burger animation sound sync. The pop sound effect was out of sync with the layered burger drop animation. The audio file was edited in Clipchamp into a short single pop, which is then triggered individually for each layer with a timed delay.
  • Background music controls. The background music originally had no way to stop it once started. A play/pause toggle was added so users remain in control of the audio at all times.
  • Mobile layout overflow. The Implementation section and Generative AI Usage Record table overflowed horizontally on small screens. Responsive CSS was adjusted so long text wraps correctly and tables fit within the available screen width on mobile devices.
