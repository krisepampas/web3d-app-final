La Boulangerie 3D Food

For this Web 3D application, I created a bakery and food themed interactive gallery using three original Blender models: a cheeseburger, a green apple and a bread basket. The application allows users to switch between models, inspect them in real time, trigger model-specific animations, change lighting, use camera viewpoints, switch wireframe mode and play audio through the web interface.

3D models

The cheeseburger, green apple and bread basket were modelled in Blender and exported as GLB files for the Three.js viewer. The cheeseburger is built from separate bun, cheese and patty objects so that the web application can animate the layers individually. The cheese was shaped with a cloth and collision workflow to make it sit more naturally on the burger.

The apple was made from a reshaped sphere with a separate stem. The bread basket is the most detailed model and uses Extra Curve Objects, Knots/Celtic Links curves, NURBS ring shapes and three separate bread objects. The original Blender files are stored in the blender_files folder, and the exported web models are stored in models.

Materials were prepared in Blender before export. The burger buns, patty, apple and bread use baked texture images so the procedural material work appears consistently in the browser. The bread basket model was also optimised with Decimate because the original file was too large for smooth web use.

Design

The interface uses warm cream, brown and gold colours to match the bakery theme. The header uses a text-based brand and strapline, while the Gallery page uses a left-side model menu and a large central 3D viewer so the selected model remains the focus. The controls are placed directly under the viewer to make wireframe mode, animation, lighting and camera views easy to find.

Bootstrap is used for the responsive grid, navigation and layout utilities, while custom CSS gives the application its own visual identity. The design also considers accessibility by using semantic HTML elements such as header, main, aside, section and footer for a clear page structure, plain text labels on every button so the function is obvious without relying on icon literacy, readable responsive layouts that adapt to phone and tablet screens, and user-controlled audio rather than autoplay so users with sensory sensitivities are not surprised by sound.

Integration

The application integrates HTML5, CSS3, Bootstrap, JavaScript, Three.js, GLTFLoader and OrbitControls. Three.js creates the WebGL scene, camera, renderer, lighting and shadow setup. GLTFLoader imports the Blender-exported GLB files, while OrbitControls allows each model to be inspected interactively.

The Gallery page integrates the 3D models with short supporting text descriptions, lighting controls, camera buttons, background music and a short burger sound effect.

The burger assembly sound was edited in Clipchamp into a short single pop sound because the original timing did not match the layered animation. In the JavaScript implementation, this short sound is triggered multiple times with timed delays during the burger animation so each layer has a separate pop effect.

Interaction

The Gallery page supports model switching between Cheeseburger, Green Apple and Bread Basket, with OrbitControls used for dragging, rotating and zooming around each loaded GLB model. Users can inspect the model structure with wireframe mode, start or stop automatic rotation, reset the camera, and use Front, Side, Top and Detail viewpoint buttons that move the camera with smooth JavaScript tweening. The interface also includes Warm, Studio and Oven Glow lighting presets, a live light intensity slider, background music controlled by a button, and model-specific animations: the burger layers drop into place, the apple rolls side to side, and the bread basket rotates.

Implementation

The application was implemented as a responsive two-page Web 3D site. The Gallery page contains the interactive Three.js scene and model controls, while this About page explains the modelling workflow, design choices, technologies, implementation, testing evidence and deeper understanding work. JavaScript handles model loading, button events, active control states, animation timing, audio playback, camera movement and UI text updates.

Rationality of technology choices. Three.js was chosen as the Web 3D format because it was taught in the labs, has wide browser support, loads Blender's native GLB export directly through GLTFLoader and has more community documentation than X3DOM. GLB was chosen over X3D because it is the modern Blender export format, produces smaller files, and matches the Three.js workflow without conversion steps. Bootstrap is used for the responsive grid and navigation utilities because it is faster than writing custom media queries from scratch, and custom CSS layered on top gives the site its own bakery visual identity rather than a default Bootstrap look. The HTML5 audio element was used for music and sound effects because it is supported natively without an extra library.

I conducted functional testing on the Gallery page to check that the model selection buttons, wireframe mode, animation button, lighting presets, camera viewpoint buttons, music button and navigation links worked as expected. I also tested the site on different screen sizes to check that the layout remained readable.

I also asked a classmate to test the website and complete a cognitive walkthrough of the main controls. This feedback helped me identify several practical issues in the interactive experience. For example, the apple rolling animation originally clipped through the large scene tray, so I reduced the tray size to give the model more space to move.

The bread basket model was also too heavy at first, especially when wireframe mode was enabled in the browser. After feedback that the page became slow and less smooth, I used Blender's Decimate modifier to reduce the model complexity while keeping a balance between visual detail and web performance.

Other feedback led to smaller interface and media improvements. Some emoji-style icons on the buttons did not display correctly, so I removed them and kept clear text labels instead. The burger animation sound was originally out of sync, so I edited the audio into a short pop sound and triggered it multiple times during the layered animation. The background music also originally started without a stop option, so I added a button state that allows users to play and pause it. Music support was also added to the About page so the audio experience remains consistent when users move away from the Gallery page.

Mobile testing also showed that the Implementation and publication section and the Generative AI Usage Record table did not fit the phone screen correctly. I adjusted the responsive CSS so long text can wrap inside its card and tables fit the available screen width on smaller devices.

Deeper Understanding

This project goes beyond the base lab tasks by replacing the lab Coca-Cola theme with an original bakery and food concept. I created three original Blender models instead of using the lab assets: a cheeseburger, a green apple and a bread basket. The models use workflows that go further than simple primitive modelling, including cloth and collision shaping for the burger cheese, procedural material node setups, UV baking, curve-based basket construction and NURBS ring details.

I also applied optimisation and export decisions for web performance. The bread basket was originally too large and became slow in the browser, especially in wireframe mode, so I used Blender's Decimate modifier to reduce the model complexity while keeping the woven basket detail readable. Baked texture images are used so the visual material work from Blender can display consistently in the Three.js viewer without making the browser recreate the full Blender shader networks.

The JavaScript implementation also extends the lab examples. The application loads and switches between multiple GLB files, normalises model scale automatically, provides wireframe inspection, and uses custom lighting presets with a live intensity slider instead of leaving lighting control as a development-only panel. The Gallery also includes Front, Side, Top and Detail viewpoint buttons that move the camera and OrbitControls target smoothly with JavaScript tweening, making model inspection more controlled than a sudden camera jump.

The model animations are also specific to the object rather than being a single generic rotation. The burger animation controls named GLB parts individually so each layer drops into place with bounce easing and timed pop sound effects, while the apple rolls side to side and the bread basket rotates to show its woven form. These interactions support the story of each food object and show a deeper understanding of how model structure, JavaScript animation and user interface controls can work together in a Web 3D application.

Implementation and publication

The completed project is organised as a Web 3D application folder containing the Gallery page, About page, JavaScript, CSS, audio files, baked texture images, exported GLB models and original Blender files. The site has been tested locally through Visual Studio Code and uploaded to the University of Sussex ITS web server for publication.

The GitHub repository link is included in the site navigation so the codebase, exported GLB models and Blender source files can be reviewed. The About page also records the modelling workflow, testing feedback and deeper understanding work required for the assignment.

The 3D models are original Blender work created for this project. External libraries and media are credited in the References section below.
